Microserview Arhitecture

The Micro service architecture is divided into three sections which are as follows:

  • Spring Framework: This is the most important component of the micro service architecture which is sectioned into various modules. We can relate each module within the core container. Different data can be accessed with the help of ‘Data Access Object’ and ‘Metadata’ through which various operations can be performed on these objects. ‘Metadata’ is something that tells about the information or behaviour about the data we are considering. It is then passed to spring container which is consisting of ‘Web’ and ‘Data Access’ parts. The major advantages of using Spring boot as a micro service due to presence of its own Apache Tomcat Server, so we need not to run or load another server in local machine. ‘Web’ is consisting of different functionality like Servlet, Web socket, Portlet etc. Similarly, ‘Data Access’ can perform functionality like Transactions, Repository, JDBC, JPA etc.
  • Integrated Development Environment: This is the section in which actual code for implementation is written using various java classes interlinked together with various packages. Spring boot uses annotation-based programming concept by which length of the program is reduced. These annotations can be used to connect different classes, interfaces, Plain Old Java Object (POJOs). Controller is the starting node from where based on the Http methods like GET, POST, PUT it will transfer its control to next class which is ‘Service’. The main code resides inside ‘ServiceImpl’ class which contain the business logic and flow. DAO Implementation is responsible for accessing the objects from variables to different databases like SQL, Mongo etc. once this is done, the correspondent values will be shown in the UI or API Client application like Postman.
  • Maven Framework: This is built and package management tool which is mainly used to bind the enterprise application project. It mainly uses ‘Project Object Model’ (pom.xml) for dependency injection for importing various functionalities. We can build the Spring project once completed to convert into a package which is Java Archive (jar). Later, this jar can be used to run the Spring boot standalone application.
